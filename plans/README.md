## Findings considered and rejected

Recorded so the next audit does not re-litigate them:

- **"Corrupt JSON in safeStorage.getItem returns raw string and crashes"**
  (`src/store/store.ts:115`) — misread: the discarded `JSON.parse(raw)` is the
  validation; a throw lands in `catch` which returns the fallback. Confirmed
  empirically (corrupt save → fresh state, no crash).
- **"README deploy steps don't warn about vite `base`"** — they do
  (`README.md:29`).
- **"CI lacks a typecheck gate"** — `bun run build` = `tsc -b && vite build`;
  typecheck is gated. (The missing TEST gate is real and folded into plan 001.)
- **Division-by-zero guard in `shouldUnlockNextStage`** — `unlockedFacts(stage)`
  cannot be empty for any reachable stage; the NaN failure mode would be "no
  unlock", not a crash. Not worth doing.
- **StrictMode double confetti in HatchScreen** — dev-only cosmetic; production
  unaffected. Not worth doing.
- **Keypad re-renders 12 buttons per keystroke** — micro-optimization; React
  re-render of 12 static elements is negligible even on a tablet. Not worth doing.
- **"Keypad should take props instead of reading the store"** — by design: the
  one-code-path input contract in `src/CLAUDE.md` has components call store
  actions directly. Not a violation.
- **Caret version ranges risk drift** — `bun.lock` + `--frozen-lockfile` in CI
  make installs reproducible; ranges only matter on explicit `bun update`. Not
  worth doing.
- **Linter/formatter/editorconfig/engines field** — marginal value for a
  single-author hobby repo with strict TS already enforced. Revisit only if a
  second contributor appears.
- **BackButton duplication across 3 screens** (`CollectionScreen.tsx`,
  `HatchScreen.tsx`, `DebugScreen.tsx`) — real but tiny; user declined a plan.
  Fine to fold into any future UI work touching those screens.

## Direction options (not planned — for the maintainer to pick up later)

Presented during the audit, none selected for planning:

1. **PWA (manifest + service worker)** — offline intent already in the repo
   (self-hosted font "gra ma działać też offline", `apple-touch-icon` present,
   no `manifest.json`); would give full-screen add-to-home-screen and real
   offline. Trade-off: SW cache invalidation complicates deploys.
2. **Save export/import** — `SaveState` is one versioned JSON blob; transferring
   progress tablet↔laptop is one export button away.
3. **Parent progress view** — DebugScreen already renders the full mastery
   table; a friendly "Postępy" screen is mostly a restyle.
4. **Reward sounds** (weakly grounded) — hatch/streak feedback peaks are
   visual-only; iOS requires AudioContext created in a user gesture.
